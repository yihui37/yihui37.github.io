模块化

2019-02-27 前端 PV:

自开始学前端以来就一直听到模块化的概念，一直在各个CLI工具下拿来即用，也没有真正去了解它是如何使用、在环境中如何编译打包的，谨以此篇记录自己对模块化的学习实践。

基本语法

export

  • export default 仅输出一个
  • export name1 输出多个时，指定具体名称

    import

  • import name 引入仅输出一个的模块
  • import {name1,name2} 引入输出多个的模块中的指定对象

开发环境如何运行

编译

Babel

由于现代浏览器还未能完全实现ES的标准语法，因此需要使用Babel进行代码编译，以兼容各大浏览器，正确运行。
如果使用Babel-loader 8.x 注意更新Babel至7.x 否则会报错
注意一下新的依赖名称即引用、配置方式

  • @babel/core
  • @babel/preset-env
  • 配置.babelrc文件

打包

Webpack

  • 配置webpack.config.js文件 webpack配置文档
  • 配置入口文件
  • 输出文件
  • 打包文件
  • loader

rollup.js

简介
  • 庞大项目的压缩打包
  • 功能单一，仅打包模块化webpack功能强大，但学习成本高
  • 参考设计原则和《Linux/Unix设计思想》
  • 工具尽量功能单一、可集成、可扩展
    安装与使用
  • 安装rollup
  • 配置类比webpack，详见rollup配置文档

众多模块化标准

发展史

  • 无模块化
  • AMD成为标准，require.js
  • 打包工具使得nodejs模块化可以被使用（common.js）
  • ES6想统一所有模块化标准
  • 目前nodejs积极支持，但浏览器尚未统一
